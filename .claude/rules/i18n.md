# i18n

13 locales via `next-i18next`, synced through Crowdin. Three namespaces:

- `common` — general UI strings
- `projects-item-data` — project descriptions
- `open-source-data` — open source contribution descriptions

`public/locales/en/` is the **source**; the other 12 are translations.

## Rules

- **Every user-facing string is a `t()` key.** Never hardcode display copy into a
  component, including copy taken from the design.
- Update the `en` source when adding keys. Do not hand-edit the 12 non-en locales — they
  come from Crowdin.
- Preserve component interpolation. `about-main-description` uses a `<0>…</0>` Chakra-link
  interpolation — keep it as a `t()` with a component; do **not** hardcode the inline
  `<a>` the design shows.
- Project/OSS structure lives in `src/components/ProjectsSection/utils.ts`.
  `src/lib/data.ts` holds only social links.

## Crowdin sync is approval-gated

**Read-only MCP queries need no approval.** Any upload / re-sync / locale deletion is
external and published — **stop and get Tyler's explicit approval first.**

For PR2 the split is decided:

- **In-repo half, automated:** new `en` source strings + key extraction. Mechanical,
  inward, reversible.
- **Upload / re-sync of the 12 non-en locales: carved out to a separate approval-gated
  follow-up.** It must not fire mid-run, and must not push to the live project before the
  redesign has Tyler's visual sign-off.

PR2 adopts the design's rewritten copy as the new `en` source, so **all 12 locales
re-translate** — not just new keys. Budget for that churn.

Known data bug the redesign copy fixes: `project-item-weatherapp-description` is currently
a copy-paste of the Cloudflare login text.
